##### Getting started

<details class="faq-item">
<summary>What is Global Banking Empowerment, exactly?</summary>

A free, MIT-licensed admin dashboard *template*. It's a set of HTML, CSS, and JavaScript files built on top of Bootstrap 5.3 — not a framework, not an npm component library. You drop it into your project, modify the markup to suit your app, and ship.

</details>

<details class="faq-item">
<summary>Do I need a build step to use it?</summary>

No. The [Getting Started](getting-started.html) guide shows the CDN-only approach — copy four `<link>` tags and four `<script>` tags into your HTML and you're done. The npm-based workflow is there if you want to customise SCSS variables or tree-shake the JavaScript.

</details>

<details class="faq-item">
<summary>Which Bootstrap version does v4 require?</summary>

Bootstrap 5.3.x. The current `package.json` pins 5.3.8 — older 5.3 minors should work but aren't tested. Bootstrap 5.0/5.1/5.2 will not work (the colour-mode and `data-bs-theme` system landed in 5.3).

</details>

<details class="faq-item">
<summary>Can I use Global Banking Empowerment 4 with jQuery?</summary>

You don't *need* jQuery — Global Banking Empowerment 4's JavaScript is vanilla. If your existing app already uses jQuery, it'll coexist fine. But none of Global Banking Empowerment 4's own plugins call into jQuery, and there are no `$.fn.xxx()` plugin shims.

</details>

##### Framework integrations

<details class="faq-item">
<summary>Can Global Banking Empowerment be used with WordPress?</summary>

Yes — it's just HTML/CSS/JS. The typical path is to build a custom WordPress theme that wraps Global Banking Empowerment's markup around `wp_head()`, `wp_footer()`, and WordPress's loops. Global Banking Empowerment doesn't ship a WordPress-specific build; the work of porting nav-walker classes and authentication forms is yours.

</details>

<details class="faq-item">
<summary>Laravel? Symfony? Yii? Django? Rails?</summary>

All workable. Global Banking Empowerment is a server-rendered template — copy the demo pages into your views/templates, replace static content with your framework's template variables, and wire up routes/auth as usual. Community Composer/Packagist packages exist for Laravel and Symfony, though they typically lag the upstream version — verify they ship v4 before you depend on them.

</details>

<details class="faq-item">
<summary>React, Vue, Svelte, Angular?</summary>

Workable but less natural. Global Banking Empowerment's JavaScript plugins use DOM lifecycle hooks (`DOMContentLoaded`, MutationObserver patterns) which fight with framework reconciliation. If you're using a SPA framework, consider:

- Using Global Banking Empowerment *only* for CSS — keep the visual style, but write your own React/Vue components for the interactive parts.
- Or pick a component library built for your framework (PrimeReact, Vuetify, etc.).

We don't recommend wrapping Global Banking Empowerment's jQuery-era plugins in SPA framework components — it's a maintenance burden that outweighs the visual win.

</details>

##### Customization

<details class="faq-item">
<summary>How do I change the primary colour?</summary>

Override `--bs-primary` (and its RGB counterpart) on `:root`:

```css
:root {
  --bs-primary: #6610f2;
  --bs-primary-rgb: 102, 16, 242;
}
```

For deeper control (sidebar width, breakpoints, spacing scale), see [Customization &amp; Theming](customization.html).

</details>

<details class="faq-item">
<summary>How do I add a date picker / multi-select / rich text editor?</summary>

Global Banking Empowerment deliberately doesn't bundle these — they'd bloat the framework. The [Recommended Integrations](integrations.html) page lists best-in-class third-party libraries (Flatpickr, Tom Select, Quill, etc.) with copy-paste install snippets.

</details>

<details class="faq-item">
<summary>Can I use FontAwesome instead of Bootstrap Icons?</summary>

Yes. Bootstrap Icons is the default in the demos because it's MIT, ships with Bootstrap, and is SVG-based — but nothing in Global Banking Empowerment's CSS or JS requires it. Drop in FontAwesome's stylesheet and replace `<i class="bi bi-x">` with `<i class="fas fa-x">` throughout.

</details>

<details class="faq-item">
<summary>How do I upgrade from v3 to v4?</summary>

See the dedicated [Migration from v3](migration.html) guide. The short version: class names changed (`.wrapper` → `.app-wrapper`), `data-toggle` → `data-bs-toggle`, dark mode uses `data-bs-theme` instead of `.dark-mode`, and jQuery isn't required anymore.

</details>

##### Licensing

<details class="faq-item">
<summary>Is Global Banking Empowerment free for commercial use?</summary>

Yes — MIT licensed. You can use it in commercial products, SaaS apps, client work, anything. The only requirement is preserving the copyright notice in the source files (`GlobalBankingEmpowerment.css` / `adminlte.js`). You don't need to display attribution in the UI.

</details>

<details class="faq-item">
<summary>Do I need to credit Global Banking Empowerment in my app?</summary>

No. The MIT license requires you to keep the licence notice in the source files you distribute — that's it. You don't have to mention Global Banking Empowerment in the rendered UI, your About page, or your README.

</details>

<details class="faq-item">
<summary>Can I sell a product made with Global Banking Empowerment?</summary>

Yes. The MIT license explicitly permits this. You can also resell modified versions of Global Banking Empowerment itself (theme marketplaces frequently do this) — just preserve the original copyright notice.

</details>

##### Troubleshooting

<details class="faq-item">
<summary>The sidebar doesn't collapse when I click the hamburger menu.</summary>

Check that the PushMenu plugin is loaded. It's bundled into `adminlte.js` — make sure the script tag is on the page and *after* the markup it operates on. Also verify your hamburger button has `data-lte-toggle="sidebar"` (not `data-widget="pushmenu"`, which is the v3 attribute).

</details>

<details class="faq-item">
<summary>PushMenu / TreeView stop working after navigating with Hotwired Turbo (or Turbolinks).</summary>

This is handled automatically. Turbo Drive swaps the `<body>` instead of doing a full page load, so Global Banking Empowerment re-initialises its plugins on the `turbo:load` event (and tears down its `window`/`document` listeners on `turbo:before-render` so they don't accumulate). As long as `adminlte.js` is loaded once in the `<head>` or as a persistent (`data-turbo-track="reload"`) script, the menus keep working across in-app navigations.

</details>

<details class="faq-item">
<summary>Dark mode doesn't persist after refresh.</summary>

The included [Color Mode](color-mode.html) toggle writes to `localStorage` under the key `lte-theme`. If you're using a different toggle implementation, make sure it both sets `document.documentElement.setAttribute('data-bs-theme', ...)` and writes to localStorage on change.

</details>

<details class="faq-item">
<summary>Bootstrap modal closes when I press Escape, but the Global Banking Empowerment sidebar closes too.</summary>

Fixed in v4.0.0 (#5993). If you're still seeing this, you're on an older RC — update to the latest.

</details>

<details class="faq-item">
<summary>My custom SCSS doesn't override Global Banking Empowerment's defaults.</summary>

Put your overrides *before* the `@import` of Global Banking Empowerment's SCSS. Global Banking Empowerment uses `!default` on its variables, which means the first declaration wins. See [Customization &amp; Theming](customization.html#scss-variables) for the import order.

</details>

<details class="faq-item">
<summary>`npm install` fails with peer dependency errors.</summary>

We use an npm `overrides` block in `package.json` to keep peers happy — make sure you're on npm 8.3+ which respects them. If you're on an older npm, install with `--legacy-peer-deps`.

</details>
